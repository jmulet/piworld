Creación de actividades

Para crear una nueva actividad acceda a ActivityTube y pulse el botón CREAR. Rellene le formulario y se le conducirá al editor integrado IDE dónde podrá editar la activitad.

Cuando se crea una actividad, se le asigna un identificador único ID. Además, se crean dos carpetas para albergar los contenidos de la actividad. Una de las carpetas se encuentra en el lado del cliente y es pública, mientras que la otra únicamente es accesible desde el lado del servidor. A partir de ahora, llamaremos a estas carpetas client/ID/ y server/ID/ respectivamente. Tendrá acceso a estas carpetas desde el IDE.

Una actividad se puede crear directamente escribiendo codigo javascript en el fichero client/ID/QProvider.js, que a su vez, debe ser comprimido en un fichero llamado client/ID/QProvider.min.js. Esta forma da la máxima flexibilidad posible aunque se hace algo pesada para quien no domine el lenguage javascript. Por este motivo, existe un modo alternativo para definir las actividades mediante la edición de pares "key"-"value" en un fichero del servidor server/ID/QProvider.json. Básicamente, mediante las instrucciones del fichero json, el servidor es capaz de generar automaticamente el fichero de código javascript y a su vez comprimirlo. Por lo tanto, cada vez que guarde el fichero json, los posibles cambios manuales efectuados en el fichero js se perderan.

Estructura del fichero QProvider.json

El código siguiente muestra la estructura básica del fichero json. Para más información sobre la sintaxi de este tipo de ficheros acuda a la pàgina JSON.
{ "activity": { "author": "jmulet", "levels": "4 ESO", "title": { "en": "Cangur test 2012 - Level 4 ESO", "ca": "Proves Cangur 2012 - Nivell 4 ESO", "es": "Pruebas Canguro 2012 - Nivel 4 ESO" }, "description": { "en": "", "ca": "", "es": "" }, "category": "MISC", "type": 0 }, "config": { "shuffleQuestions": true, "shuffleOptions": true, "browsable": false, "timeout": 60, "allowerrors": 2, "canAskTheory": false, "canAskHelp": false }, "questions": [ { "title": "", "parameters": { "expr": "self.database.random()" }, "repeat": 10, "type": "free.multiplechoice", "serverData": [{"var":"database", "file": "server-data.json"}], "tips": "" } ] }

La propiedad "activity" define la meta-información de la actividad así como la categoria (ver el listado de categorias) y el tipo de actividad ("type" 0 normal, 1 compuesta). La propiedad "config" permite la configuración de la actividad. Utilitze un timeout >0 en segundos para realizar la actividad con cuenta atrás. Por último, la propiedad "questions" define una lista de preguntas. En la propiedad "type" se define el tipo de actividad y "repeat" indica cuantas veces debe repetirse.
